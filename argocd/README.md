# ArgoCD GitOps Deployment

This directory contains the configuration necessary to deploy the MERN application continuously using ArgoCD. With this setup, your Kubernetes cluster will automatically synchronize with the manifests located in your Git repository's `k8s/` directory.

## 1. Install ArgoCD
If you haven't installed ArgoCD on your Kubernetes cluster yet, run the following commands:
```bash
kubectl create namespace argocd
kubectl apply -n argocd -f https://raw.githubusercontent.com/argoproj/argo-cd/stable/manifests/install.yaml
```

## 2. Access ArgoCD UI
To access the ArgoCD web interface, port-forward the ArgoCD server:
```bash
kubectl port-forward svc/argocd-server -n argocd 8080:443
```
You can now access the UI at [https://localhost:8080](https://localhost:8080).

## 3. Login
Retrieve the initial admin password generated by ArgoCD:
```bash
kubectl -n argocd get secret argocd-initial-admin-secret -o jsonpath="{.data.password}" | base64 -d
```
Log in to the UI using the username `admin` and the password retrieved above.

## 4. Apply Application Configuration
Before applying, ensure you have updated the `repoURL` in `application.yaml` to point to your actual Git repository.

Apply the ArgoCD Application manifest:
```bash
kubectl apply -f argocd/application.yaml
```

## 5. Verify Synchronization
ArgoCD will immediately detect the application and begin syncing the `k8s/` manifests to the cluster.
Check the status of the ArgoCD application:
```bash
kubectl get applications -n argocd
# OR using the ArgoCD CLI:
argocd app get mern-shoppilot
```

## 6. Check Application Health
Verify that the pods, services, and deployments are running correctly in the `shoppilot` namespace:
```bash
kubectl get deployments -n shoppilot
kubectl get pods -n shoppilot
```

## 7. Trigger Deployment via Git
To test the GitOps workflow:
1. Make a change to one of the manifests in the `k8s/` directory (e.g., increase the backend replicas).
2. Commit and push the changes to your Git repository.
3. Because the `syncPolicy` is configured with `automated`, `prune`, and `selfHeal`, ArgoCD will automatically detect the commit and synchronize the state within a few minutes.

You can forcefully trigger a sync via CLI if you don't want to wait:
```bash
argocd app sync mern-shoppilot
```

## 8. View History
To view the deployment history and see which Git hashes are currently deployed:
```bash
argocd app history mern-shoppilot
```
